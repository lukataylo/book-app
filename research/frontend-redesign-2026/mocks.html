Editorial learning path
Renders Read → Remember → Act as one spatial journey per book; a single glowing "next" node. Stickiest archetype — but the design bet is keeping it serif-and-ink editorial, not Duolingo-cute (ink nodes, printed-almanac feel, no cartoon mascot). Highest build effort; new path renderer + node state.
Transition: node → lesson zoom; completion = restrained KeyframeAnimator seal stamp (demo ③).

How to read these: the four phones are alternative home screens; the three stages are the transitions that carry you into sub-screens. My recommendation is Option 1 (Today dashboard) as the new home, with the Option 5 zoom transitions applied app-wide, keeping the current shelf as a Browse tab — highest retention payoff for medium effort, since the data already exists. Option 4 is the higher-ceiling, higher-risk bet if you want to go all-in on stickiness. Tell me which to take to a working SwiftUI prototype and I'll build it on claude/frontend-ui-redesign-7e6lh6.
